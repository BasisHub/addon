2. WHAT IS NEW IN THIS RELEASE(top of page)

Version 15.10 takes advantage of the technological advancements provided by Barista® and BBj® to deliver an innovative software package known for its structural quality. Together with Barista, AddonSoftware offers a leap forward in the efficiency of software customization and the creation of vertical applications. AddonSoftware continues its long history of constant improvement - additional features, increased functionality, and updated technology - in this release.
  • Preview for version 16.00
    • Accounting
      • More defaults have been added to the New Customer Defaults to facilitate adding new customers. Launch the Accounts Receivable => Maintenance => Parameters => New Customer Defaults task and enter all the required new fields.
    • Distribution
      • Converted the Purchase Order from BBjForm-based document to a Jasper report and added interface with Report Control.
    • Utilities
      • Altered the AIUW Upgrade Wizard to always suggest an empty, version-specific test location for data files.
  • New for version 15.00
    • All forms and reports are now launchable in BUI.
    • Addon's tabbed forms enjoy an updated look with the addition of icons to the tab text.
    • New BBj methods have been incorporated into Addon's table open and error handling routines for error 0's to show the user and host that has the file or record locked.
    • Addon Dashboard
      • The GL Income Comparison widget now has drilldown capability. Click on a period revenue column, and the chart changes to show the revenue for that period by account. Then click on an account column to drill further and see a grid showing the transactions for the account for the period.
      • New widget in the Accounting category provides Company level aging with drilldown to show top customers for the selected aging category.
      • New Manufacturing tab with a widget that shows Work Orders linked to Sales Orders; includes filters for both Work Order status and Sales Order Status.
      • The Top Salesreps widgets have new drilldowns that present a grid detailing YTD and Prior Year Sales by Product Type.
      • Fast access to the dashboard via a new top-level menu.
      • Optimized SPROCs and new Sales Analysis totals tables improve load/display times for Sales Analysis data.
      • Month and Year filters have been added to the AR Drilldown Sales Report widget.
      • Uses the new setChartRangeAxisToCurrency() method on applicable widgets.
    • Report Control
      • New fields in the Report Control Document table allow association of Barista Email Account with individual documents, as well as default From, Reply-to, Subject and Message to streamline entry of Report Control Recipient records.
      • Customer and Vendor Report Control tables consolidated into single, Report Control Recipients table. New table also accommodates "other" type recipient for stand-alone reports.
      • New Report Control custom object makes it easy to fax/email enable documents.
    • Accounting
      • Embedded dashboard widget on Customer Master form shows customer's aged balance, dynamically switching from a pie chart to a bar chart to show credit balances.
      • SPROC code optimizations boost the speed of the AR Drilldown Sales Report.
      • AR Statements and Simple Invoices are enhanced, so in addition to the current Master copy showing all statements/invoices in the batch run, there is also now a Print copy containing only those documents that need to be printed. Closing the Master copy prompts the user to forward designated documents to the document queue, ensuring that documents don't go into the queue without approval.
      • When generating a statement from within the Customer form, the 'Send via Report Control' checkbox will only be enabled if the customer has a report control record specifying email or fax transmission. In addition, closing the statement viewer will prompt the user to forward the statement to the document queue, ensuring that documents don't go into the queue without approval.
      • The Balance and Sales drilldowns on the Customer Master have been improved to include both OP and AR invoices, with hyperlinks on the invoice number to launch either OP Historical Invoice Inquiry, or Barista's Document Inquiry, filtered on the AR Invoice Number. New fields showing the last aging criteria for this customer have been added to the Customer form, along with drilldowns for each aged balance field. The new drilldowns make use of the aging criteria to show the invoices associated with each aging bucket.
      • The AR Invoice header file now contains a balance field, so tasks no longer need to read through all of the invoice detail in order to separate zero balance from unpaid invoices.
      • Accounts Payable Export 1099 Data to Text has been updated for 2014 export to Account Ability.
    • Distribution
      • Realtime Processing in OP; Order/Invoice Entry have undergone a revision to eliminate the two-step process of entering data into one set of files, then transferring the data to the transaction history files after the register/update. All records are now written directly to the transaction history files, using new fields that carry the record status and creation/modification stamps to direct data entry, registers, updates, and queries. New Custom query leverages the structural modifications, and allows users to query on open orders only, open orders and invoices, or all orders and invoices.
      • New custom query improves performance when creating a Duplicate or Credit Memo in Order/Invoice Entry. Hyperlinks in the query grid let the user launch a secondary query on invoice detail, or launch the Invoice History Inquiry form.
      • Invoicing has been enhanced so that, in addition to the current Master copy showing all invoices in the batch run, there is also now a Print copy containing only those invoices that need to be printed. The Print Now button in Invoice Entry launches a new form so the user can specify a message for the on-demand invoice, as well as select whether or not to process the invoice according to report control settings. Both batch (Master) and on-demand invoices will prompt the user to update the print file. Only when the user chooses to do the update are print flags set and applicable invoices added to the document processing queue, ensuring that documents don't go into the queue without approval.
      • The Historical Invoice is now a Jasper report, giving it the same format as regular invoices, and also providing email capability.
      • Added product type and related index to IV Warehouse Master table to improve performance on the Stock Status Report.
    • Manufacturing
      • --
    • Administration
      • --
    • Utilities
      • New Auto-conversion Utility uses a config file to record the conversion steps necessary when moving from one version to another. The utility runs automatically when launching Addon after an upgrade, ensuring the proper post-upgrade conversions are accomplished before running the application.
    • Testing
      • This version passes all QF-Test suites.

  • For a list of continued improvements in all modules, refer to the Readme/Relnotes and Fixes link on the BASIS product download page.
3. KNOWN ISSUES(top of page)
  • BBj version 15.10 or higher and the Barista Application Framework version 15.10 or higher are both required to run AddonSoftware version 15.10. Earlier versions may produce unexpected results
  • When running under Linux, the KDE desktop environment must be used.
4. SUPPORT(top of page)

If you purchased your AddonSoftware product from a distributor, reseller, or OEM, please contact them for technical support. They will answer your question or contact us on your behalf.

Technical support for version 15.10 is available via e-mail at addon-developer@addonsoftware.com.

When submitting a technical support question, please provide the following information:
  1. The product serial number.
  2. The name of the product and the product's version number.
  3. A complete description of the problem, including the product error and the TCB(10) value, if available.
  4. A screen shot of the problem.
Other pertinent information includes:
  1. The operating system name, JRE, level and version.
  2. Is the error consistently reproducible, or is it sporadic?
  3. Is the error isolated to a specific machine or user?
  4. Is this a new installation or has the system been in place and operating normally?
  5. Is the problem network related/specific?
