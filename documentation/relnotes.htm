2. WHAT IS NEW IN THIS RELEASE(top of page)

Version 17.12 takes advantage of the technological advancements provided by Barista® and BBj® to deliver an innovative software package known for its structural quality. Together with Barista, AddonSoftware offers a leap forward in the efficiency of software customization and the creation of vertical applications. AddonSoftware continues its history of constant improvement - additional features, increased functionality, and updated technology - in this release.
  • Preview for version 18.00
    • General
      • Implemented previous structural changes in Addon Company and Report Control files so logos for documents such as Invoices, Statements, and Purchase Orders can now use a document-specific logo, a firm-specific logo, or the default logo specified in the config file.
      • Added Import Recipients additional option to the Report Control Recipients form to facilitate a bulk import of customers and/or vendors.
    • Accounting
      • Implemented previous structural changes to allow 1024-character comments in the AP Invoice Header, Vendor Master, Cash Receipts Header/Detail/GL Distribution, Simple Invoice Header, Customer Master, and GL Account Master. In some cases the new comment replaces legacy 80-character comment lines, and in others new comment fields have been added. See the readme for the detailed list of files.
      • Added historical inquiry forms for AP Invoices, Cash Receipts, and Simple Invoices.
      • New Salesperson Call List report generates a list of customers with overdue balances to use as a guide for salesperson calls.
      • Bank Reconciliation allows a Current Statement Date in a locked period, as long as the date is in the prior, current, or next fiscal year.
      • Bank Reconciliation Check Detail and Other Transaction grids allow multiple selections, and include a context menu to quickly change the Code on the selected rows.
    • Distribution
      • Added a new option to Order Entry to automatically generate a Work Order for an item that has a Bill of Materials, and for which the quantity currently available is less than the order ship quantity.
      • Implemented previous structural changes to allow 1024-character comments in the Order/Invoice Header and Detail (detail comments print on Pick Lists and Invoices), OP Standard Messages, Purchase Requisition Detail, Purchase Order Detail, Purchase Order Receipt Detail, Purchase Order Invoice Detail (detail comments print on Purchase Orders), and PO Standard Messages. In some cases the new comment replaces legacy 80-character comment lines, and in others new comment fields have been added. See the readme for the detailed list of files.
      • Added historical inquiry form for Purchase Receipts.
      • Enhanced Invoice History Inquiry to allow view/edit of header-level comments, and view of detail comments.
      • Added default Beginning and Ending Dates to the Sales Tax Report.
      • Reformatted the Monthly Sales Report into columnar output consistent with the Export Format option available on several other reports.
      • Purchase Order Invoice Entry defaults the receiver to the first un-billed receiver for the entered PO.
    • Manufacturing
      • Added "Date Effective After" and "Date Obsolete Before" filters to the Bill of Materials Operations Where Used report.
  • New for version 17.00
    • General
      • Barista enhancements to Expresso bring a suite of new queries to replace and improve on legacy Addon's SpeedSearch application.
      • New MDI backgrounds can be displayed for each company given Barista enhancements to the Firms table.
      • Enhanced Expresso lookups for Customer, Item, and Vendor are now used throughout the application.
      • Increased zoom and set 'fit to width' improve readability in the Report Viewer.
      • Creation of embedded widgets is now managed by Barista for better security.
      • The Digital Dashboard's GL Income Account Totals grid widget incorporates a new BBj dashboard method to size column widths based on the contents of the column headings and the first few rows of data.
      • A new license agreement form is presented when you launch Addon, giving the end user (not just the VAR/installer) a chance to read and accept the agreement. The form will continue to appear with each launch until the user selects the Accept option and clicks OK.
    • Accounting
      • Each fiscal year now has it's own separate fiscal calendar rather than sharing a single fiscal calendar for all fiscal years. This better accommodates companies with fiscal calendars that vary from year to year.
      • General Ledger Account Summary information is now retained for all fiscal years instead of for just the current, prior and next fiscal years.
      • A new "inactive" field has been added to the Customer, Vendor, and GL Account master tables. The associated queries have pre-defined filters to view active or all records. Related reports include an option to suppress reporting inactive records. Attempting to use an inactive customer, vendor, or GL Account results in a warning, and may be prevented in some instances if appropriate. Marking a particular record as inactive does not alter data already in the system.
      • The GL Journal Update now writes to the Bank Rec Other table when Bank Rec is installed and the update processes an entry that matches one of the listed Bank Accounts, doing away with the need to enter these transactions twice.
      • Added password override to GL Update Beginning Balances task that allows updating current year beginning balance even though the prior year is closed.
      • Accounts Payable Export 1099 Data to Text has been updated for 2016 export to Account Ability.
      • The Accounts Payable check format (Check/Stub/Stub, Stub/Check/Stub, etc.) is now stored in the parameters and appears as the default on the Check Print form.
      • The GL Transaction History report has a new filter to report only the postings for a specific audit number.
    • Distribution
      • A new "inactive" field has been added to the Inventory Item master table. The associated query has a pre-defined filter to view active or all items. Related reports include an option to suppress reporting inactive items. Attempting to use an inactive item results in a warning, and may be prevented in some instances if appropriate. Marking an item as inactive does not alter data already in the system.
      • Enhanced Sales Analysis forms carry embedded bar charts to provide a visual representation of the sales, cost, and units.
      • Combined credit status and credit hold/release messages to provide more information in OP Order Entry and Invoice Entry.
      • Custom Inventory Item lookup has been enhanced to include a small image of the item selected in the lookup grid.
      • Users can now enter Beginning and Ending Receipt Dates for the PO Receipt History By Item report so it can be filtered by date.
      • You can now look up an historical invoice using just the invoice number, rather than needing both invoice and customer numbers.
    • Manufacturing
      • Bills of Material are tied to Items so if an Item is marked inactive, the corresponding Bill is inactive as well.
      • Load Balance Inquiry now uses an embedded bar chart widget for enhanced functionality.
    • Testing
      • This version passes all QF-Test suites.

  • For a list of continued improvements in all modules, refer to the Readme/Relnotes and Fixes link on the BASIS product download page.
3. KNOWN ISSUES(top of page)
  • BBj version 17.12 or higher and the Barista Application Framework version 17.12 or higher are both required to run AddonSoftware version 17.12. Earlier versions may produce unexpected results
  • When running under Linux, the KDE desktop environment must be used.
4. SUPPORT(top of page)

If you purchased your AddonSoftware product from a distributor, reseller, or OEM, please contact them for technical support. They will answer your question or contact us on your behalf.

Technical support for version 17.12 is available via e-mail at addon-developer@addonsoftware.com.

When submitting a technical support question, please provide the following information:
  1. The product serial number.
  2. The name of the product and the product's version number.
  3. A complete description of the problem, including the product error and the TCB(10) value, if available.
  4. A screen shot of the problem.
Other pertinent information includes:
  1. The operating system name, JRE, level and version.
  2. Is the error consistently reproducible, or is it sporadic?
  3. Is the error isolated to a specific machine or user?
  4. Is this a new installation or has the system been in place and operating normally?
  5. Is the problem network related/specific?
