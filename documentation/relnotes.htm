2. WHAT IS NEW IN THIS RELEASE(top of page)

Version 16.20 takes advantage of the technological advancements provided by Barista® and BBj® to deliver an innovative software package known for its structural quality. Together with Barista, AddonSoftware offers a leap forward in the efficiency of software customization and the creation of vertical applications. AddonSoftware continues its 25-year history of constant improvement - additional features, increased functionality, and updated technology - in this release.
  • Preview for version 17.00
    • Increased zoom and set 'fit to width' to improve readability in the Report Viewer.
    • Creation of embedded widgets is now managed by Barista so the connection string is not exposed in AddonSoftware code.
    • The Digital Dashboard's GL Income Account Totals grid widget incorporates a new BBj dashboard method to size column widths based on the contents of the column headings and the first few rows of data.
    • Custom Inventory Item lookup has been enhanced to include a small image of the item selected in the lookup grid.
    • A new license agreement form is presented when you launch Addon, giving the end user (not just the VAR/installer) a chance to read and accept the agreement.
    • The new license agreement form has been enhanced to include OK/Cancel buttons for more intuitive operation. The form will continue to appear with each launch until the user selects the Accept option and clicks OK.
  • New for version 16.10
    • The all new graphical Payroll module joins AddonSoftware's Accounting, Distribution, and Manufacturing bundles. See the Payroll relnotes and Fixed Issues for more information.
  • New for version 16.00
    • GoldMine CRM - AddonSoftware now integrates with GoldMine
      • Load Addon data into GoldMine database.
      • Import GoldMine data into Addon's database.
      • View selected Addon forms/queries from within GoldMine's GM+Views.
      • Launch GoldMine directly from within Addon.
    • Accounting
      • Enhanced the Bank Reconciliation interface from Cash Receipts and Invoice Entry to be more flexible in recording deposits, making it easier to reconcile to bank statements. New Deposit form attached to Cash Receipts gives the user control over which receipts will constitute a deposit. Deposits are automatically created by date and cash receipts code for Invoice Entry cash sales. Registers show Deposit ID's, and Cash Receipts register includes a recap by Deposit.
      • Increased the maximum number of GL Distributions allowed per invoice in AP Invoice Entry from 99 to 999.
      • More defaults have been added to the New Customer Defaults to facilitate adding new customers. Launch the Accounts Receivable => Maintenance => Parameters => New Customer Defaults task and enter all the required new fields.
      • The AR Drilldown Sales report has been enhanced to launch each hyperlinked report in a separate tab in the Jasper viewer.
      • New buttons on the Customer Master launch queries for Quotations, Orders, and Invoices.
      • Accounts Payable Export 1099 Data to Text has been updated for 2015 export to Account Ability.
    • Distribution
      • Converted the Purchase Requisition from a BBjForm-based document to a Jasper report.
      • Converted the Purchase Order from a BBjForm-based document to a Jasper report and added interface with Report Control.
      • Converted the Pick List/Quotation from a BBjForm-based document to a Jasper report; Quotations interface with Report Control.
      • New button on the Inventory Item Master provides quick access to a query showing item availability across all warehouses.
    • Manufacturing
      • Converted the Shop Floor Pick List from a BBjForm-based document to a Jasper report.
    • Utilities
      • Altered the AIUW Upgrade Wizard to always suggest an empty, version-specific test location for data files.
      • Boosted performance in the Copy Data from Firm to Firm and Clear Firm from System utilities by adding a checkbox to control if/when the record counts for the specified firm should be calculated. Added logging.
    • Testing
      • This version passes all QF-Test suites.

  • For a list of continued improvements in all modules, refer to the Readme/Relnotes and Fixes link on the BASIS product download page.
3. KNOWN ISSUES(top of page)
  • BBj version 16.20 or higher and the Barista Application Framework version 16.20 or higher are both required to run AddonSoftware version 16.20. Earlier versions may produce unexpected results
  • When running under Linux, the KDE desktop environment must be used.
4. SUPPORT(top of page)

If you purchased your AddonSoftware product from a distributor, reseller, or OEM, please contact them for technical support. They will answer your question or contact us on your behalf.

Technical support for version 16.20 is available via e-mail at addon-developer@addonsoftware.com.

When submitting a technical support question, please provide the following information:
  1. The product serial number.
  2. The name of the product and the product's version number.
  3. A complete description of the problem, including the product error and the TCB(10) value, if available.
  4. A screen shot of the problem.
Other pertinent information includes:
  1. The operating system name, JRE, level and version.
  2. Is the error consistently reproducible, or is it sporadic?
  3. Is the error isolated to a specific machine or user?
  4. Is this a new installation or has the system been in place and operating normally?
  5. Is the problem network related/specific?
