2. WHAT IS NEW IN THIS RELEASE(top of page)

Version 14.21 takes advantage of the technological advancements provided by Barista® and BBj® to deliver an innovative software package known for its structural quality. Together with Barista, AddonSoftware offers a leap forward in the efficiency of software customization and the creation of vertical applications. AddonSoftware continues its 25-year history of constant improvement - additional features, increased functionality, and updated technology - in this release.
  • Preview for version 15.00
    • All forms and reports are now launchable in BUI.
    • New BBj methods have been incorporated into Addon's table open and error handling routines for error 0's to show the user and host that has the file or record locked.
    • Addon Dashboard
      • ** The GL Income Comparison widget now has drilldown capability. Click on a period revenue column, and the chart changes to show the revenue for that period by account. Then click on an account column to drill further and see a grid showing the transactions for the account for the period.
      • Now uses the setChartRangeAxisToCurrency() method on applicable widgets. This method uses the current locale's currency formatter to work in various countries.
      • New Manufacturing tab with a widget that shows Work Orders linked to Sales Orders; includes filters for both Work Order status and Sales Order Status.
      • Fast access to the dashboard via a new top-level menu.
      • Optimized SPROCs improve load/display times for Sales Analysis data.
      • Month and Year filters have been added to the AR Drilldown Sales Report widget.
    • Accounting
      • ** Accounts Payable Export 1099 Data to Text has been updated for 2014 export to Account Ability
      • AR Simple Invoices and Statements now generate Print copy as well as the Master copy. The Print copy contains only those invoices/statements that need to be printed. Closing the Master copy will prompt the user to forward designated documents to the document queue, ensuring that documents don't go into the queue without approval.
      • AR On-demand Statements (launched from the Customer form) also prompt the user before forwarding the statement to the document queue.
      • Enhanced drilldowns on the Customer Master
        • The Balance drilldown now runs an enhanced custom query to view outstanding AR Invoices, showing the original invoice amount as well as current balance, with down-totals. New link on invoice number launches custom query into the invoice payment transactions.
        • Four additional drilldowns are available on the MTD, YTD, Prior Year, and Future sales fields if Order Processing is installed. These drilldowns show sales for the selected time frame, and a new link on the invoice number launches the OP Historical Invoice form.
      • Embedded dashboard widget on Customer Master form shows customer's aged balance, dynamically switching from a pie chart to a bar chart to show credit balances.
      • SPROC code optimizations boost the speed of the AR Drilldown Sales Report.
    • Distribution
      • The Duplicate/Credit Memo query in Order and Invoice Entry has been replaced with a new custom query that runs much faster since it only shows data from the historical invoice header table. New hyperlinks in the query grid let the user launch a secondary query on invoice detail, or launch the Invoice History Inquiry form.
      • OP Invoicing has been enhanced in the following ways:
        • In addition to the current Master copy showing all invoices in the batch run, there is also now a Print copy containing only those invoices that need to be printed.
        • The Print Now button in Invoice Entry launches a new form so the user can specify a message for the on-demand invoice, as well as select whether or not to process the invoice according to report control settings.
        • Both batch (Master) and on-demand invoices will prompt the user to update the print file. Only when the user chooses to do the update are print flags set and applicable invoices added to the document processing queue.

3. KNOWN ISSUES(top of page)
  • BBj version 14.21 or higher and the Barista Application Framework version 14.21 or higher are both required to run AddonSoftware version 14.21. Earlier versions may produce unexpected results
  • When running under Linux, the KDE desktop environment must be used.
4. SUPPORT(top of page)

If you purchased your AddonSoftware product from a distributor, reseller, or OEM, please contact them for technical support. They will answer your question or contact us on your behalf.

Technical support for version 14.21 is available via e-mail at addon-developer@addonsoftware.com.

When submitting a technical support question, please provide the following information:
  1. The product serial number.
  2. The name of the product and the product's version number.
  3. A complete description of the problem, including the product error and the TCB(10) value, if available.
  4. A screen shot of the problem.
Other pertinent information includes:
  1. The operating system name, JRE, level and version.
  2. Is the error consistently reproducible, or is it sporadic?
  3. Is the error isolated to a specific machine or user?
  4. Is this a new installation or has the system been in place and operating normally?
  5. Is the problem network related/specific?
